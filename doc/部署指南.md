## 一、部署架构概览

### 1.1 单服务器架构（本地开发 / 小规模部署）

```
┌─────────────────────────────────────────────────────────┐

└───────────────────────┬─────────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────────┐
│              HertzBeat 监控平台 (端口 1157)               │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ ┌────────────┐  │
│  │ 项目管理 │ │ 监控管理  │ │ 告警管理  │ │ 日志检索    │  │
```

### 组件清单

| 组件 | 版本 | 端口 | 用途 |
|------|------|------|------|
| HertzBeat | 自定义构建 | 1157/1158 | 监控告警平台主服务 |
| MySQL | 8.0 | 3306 | 元数据存储 |
| Grafana Loki | latest | 3100 | 日志聚合存储 |
| Promtail | latest | 9080 | 日志采集代理 |
| Grafana | latest | 3000 | 日志可视化面板 |

## 二、本地开发环境部署

### 2.1 前置条件安装

#### 2.1.1 安装 JDK 17+

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-17-jdk -y
java -version

# CentOS/RHEL
sudo yum install java-17-openjdk-devel -y
java -version

# macOS
brew install openjdk@17
```

#### 2.1.2 安装 Maven 3.8+

```bash
# Ubuntu/Debian
sudo apt install maven -y
mvn -version

# macOS
brew install maven
```

#### 2.1.3 安装 Node.js 18+（前端构建需要）

```bash
# 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
node -v
npm -v
```

#### 2.1.4 安装 Docker 和 Docker Compose

```bash
# Ubuntu
sudo apt update
sudo apt install docker.io docker-compose-plugin -y
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker $USER

# CentOS
sudo yum install docker-ce docker-compose-plugin -y
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
docker compose version
```

### 2.2 编译 HertzBeat 二次开发版本

#### 2.2.1 进入源码目录

```bash
cd e:/2_git_code/system-monitor-alert/hertzbeat
```

#### 2.2.2 执行 Maven 编译打包

```bash
# 跳过测试编译打包（首次编译时间较长）
mvn clean package -DskipTests

# 如果编译失败，可尝试指定 JDK 版本
# export JAVA_HOME=/path/to/jdk17
# mvn clean package -DskipTests
```

#### 2.2.3 确认构建产物

```bash
# 构建产物位于 hertzbeat-startup/target/ 目录
ls -la hertzbeat-startup/target/
# 应看到 hertzbeat-xxx.jar 文件
```

### 2.3 启动基础设施（MySQL + Loki + Promtail + Grafana）

#### 2.3.1 进入项目根目录

```bash
cd e:/2_git_code/system-monitor-alert
```

#### 2.3.2 创建必要的数据目录

```bash
mkdir -p data/loki
mkdir -p data/promtail
mkdir -p data/logs
mkdir -p data/grafana
mkdir -p data/mysql
```

#### 2.3.3 启动 Docker Compose 服务

```bash
# 启动所有基础设施服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f
```

#### 2.3.4 验证基础设施服务

```bash
# 验证 MySQL
docker exec mysql mysql -uhertzbeat -phertzbeat123 -e "SELECT 1"

# 验证 Loki
curl http://localhost:3100/ready

# 验证 Grafana
curl http://localhost:3000/api/health
```

### 2.4 配置 HertzBeat 连接 MySQL

#### 2.4.1 创建生产环境配置文件

```bash
cp hertzbeat/hertzbeat-startup/src/main/resources/application.yml \
   hertzbeat/hertzbeat-startup/src/main/resources/application-prod.yml
```

#### 2.4.2 修改数据库配置

编辑 `application-prod.yml`，找到 `spring.datasource` 部分，修改为：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: hertzbeat
    password: hertzbeat123
    url: jdbc:mysql://localhost:3306/hertzbeat?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    hikari:
      max-lifetime: 120000

  jpa:
    show-sql: false
    database: mysql
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQLDialect
        format_sql: true
```

#### 2.4.3 添加 Loki 配置

在 `application-prod.yml` 末尾添加：

```yaml
hertzbeat:
  loki:
    url: http://localhost:3100
```

### 2.5 启动 HertzBeat

#### 2.5.1 方式一：JAR 包启动

```bash
cd hertzbeat/hertzbeat-startup/target

# 前台启动（调试用）
java -jar hertzbeat-startup-*.jar --spring.profiles.active=prod

# 后台启动
nohup java -jar hertzbeat-startup-*.jar --spring.profiles.active=prod > ../logs/hertzbeat.log 2>&1 &

# 查看启动日志
tail -f ../logs/hertzbeat.log
```

#### 2.5.2 方式二：Maven 启动（开发调试用）

```bash
cd hertzbeat

# 使用 Maven 启动
mvn spring-boot:run -pl hertzbeat-startup -Dspring-boot.run.profiles=prod
```

#### 2.5.3 验证启动成功

```bash
# 检查端口监听
netstat -tlnp | grep 1157

# 访问健康检查接口
curl http://localhost:1157/actuator/health

# 浏览器访问
# http://localhost:1157
# 默认账号：admin / hertzbeat
```

### 2.6 配置 Grafana 连接 Loki

#### 2.6.1 登录 Grafana

```
地址：http://localhost:3000
账号：admin
密码：admin
```

#### 2.6.2 添加 Loki 数据源

1. 进入 **Configuration → Data Sources → Add data source**
2. 选择 **Loki**
3. URL 填写：`http://loki:3100`（Docker 网络内）或 `http://localhost:3100`（宿主机）
4. 点击 **Save & Test**，显示 "Data source connected" 即成功

#### 2.6.3 创建日志查询面板

1. 进入 **Explore** 页面
2. 选择 Loki 数据源
3. 输入 LogQL 查询，如：`{job="app-logs"}`
4. 可按 project、service、level 标签过滤

## 三、政务外网服务器部署

### 3.1 部署前准备

#### 3.1.1 服务器环境检查

```bash
# 检查操作系统版本
cat /etc/os-release

# 检查 CPU 和内存

# 检查磁盘空间

# 检查端口占用
netstat -tlnp | grep -E '1157|3306|3100|3000|9080'
```

#### 3.1.2 安装 Docker 和 Docker Compose

```bash
# CentOS 7 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
docker compose version
```

#### 3.1.3 配置 Docker 镜像加速（政务外网可能需要）

```bash

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://registry.docker-cn.com"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
EOF

sudo systemctl daemon-reload
sudo systemctl restart docker
```

#### 3.1.4 政务外网离线部署准备（如无法访问外网）

#### 3.3.2 修改 Docker Compose 配置

编辑 `/opt/hertzbeat/docker-compose.yml`：

```yaml
version: "3.8"

services:
  loki:
    image: grafana/loki:latest
    container_name: loki
    ports:
      - "3100:3100"
    volumes:
      - ./data/loki:/loki
    command: -config.file=/etc/loki/local-config.yaml
    restart: unless-stopped
    networks:
      - monitor-net

  promtail:
    image: grafana/promtail:latest
    container_name: promtail
    volumes:
      - ./data/promtail/config.yml:/etc/promtail/config.yml
      - ./data/logs:/var/log/app
    command: -config.file=/etc/promtail/config.yml
    depends_on:
      - loki
    restart: unless-stopped
    networks:
      - monitor-net

  grafana:
    image: grafana/grafana:latest
    container_name: grafana
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=<修改Grafana密码>
    volumes:
      - ./data/grafana:/var/lib/grafana
    depends_on:
      - loki
    restart: unless-stopped
    networks:
      - monitor-net

  mysql:
    image: mysql:8.0
    container_name: mysql
    ports:
      - "3306:3306"
    environment:
      - MYSQL_ROOT_PASSWORD=<修改Root密码>
      - MYSQL_DATABASE=hertzbeat
      - MYSQL_USER=hertzbeat
      - MYSQL_PASSWORD=<修改HertzBeat数据库密码>
    volumes:
      - ./data/mysql:/var/lib/mysql
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    restart: unless-stopped
    networks:
      - monitor-net

networks:
  monitor-net:
    driver: bridge
```

#### 3.3.3 修改 Promtail 配置

编辑 `/opt/hertzbeat/data/promtail/config.yml`：


### 3.4 启动服务

#### 3.4.1 启动基础设施

```bash
cd /opt/hertzbeat

# 启动 MySQL + Loki + Promtail + Grafana
docker compose up -d

# 等待 MySQL 初始化完成（首次启动约30秒）
docker compose logs -f mysql
# 看到 "ready for connections" 即完成

# 验证所有服务
docker compose ps
```

#### 3.4.2 创建 HertzBeat Systemd 服务

```bash
sudo tee /etc/systemd/system/hertzbeat.service <<-'EOF'
[Unit]
Description=HertzBeat Monitor Platform
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/hertzbeat
ExecStart=/usr/bin/java -jar /opt/hertzbeat/bin/hertzbeat.jar \
  --spring.config.additional-location=/opt/hertzbeat/config/ \
  --spring.profiles.active=prod
```

## 四、多服务器网络策略与端口规划

### 4.1 核心结论

**可以监控和采集多台服务器上的服务与日志。** HertzBeat 采用**主动采集**模式，从监控服务器主动连接各应用服务器上的服务进行指标采集；日志通过**Promtail 代理**在各应用服务器本地采集后推送到集中式 Loki，或通过 **Logback Loki Appender** 由应用直发到 Loki。

**必须开通服务器间的网络策略**，否则监控和日志采集均无法工作。

### 4.2 数据流向与端口需求总览

```
  数据流方向                          协议/端口              是否必须
  ──────────                          ────────              ────────
  ① 浏览器 → HertzBeat               TCP/1157              是（控制台访问）
  ② 浏览器 → Grafana                 TCP/3000              是（日志面板）
  ③ HertzBeat → 应用服务(Actuator)   TCP/应用端口(如8080)   是（SpringBoot监控）
  ④ HertzBeat → 应用服务(HTTP)       TCP/应用端口(如8080)   是（HTTP探针监控）
  ⑤ HertzBeat → 应用服务(TCP端口)    TCP/目标端口           是（端口存活监控）
  ⑥ HertzBeat → 应用服务器(SSH)      TCP/22                可选（进程/OS监控）
  ⑦ HertzBeat → 应用服务(JMX)        TCP/自定义(如9999)     可选（JVM深度监控）
  ⑧ Promtail → Loki                  TCP/3100              是（日志推送）
  ⑨ 应用(LokiAppender) → Loki        TCP/3100              是（日志直发）
  ⑩ HertzBeat → 企微/钉钉/飞书       HTTPS/443             可选（告警通知）
  ⑪ HertzBeat → SMTP邮件服务器       TCP/465或25            可选（邮件通知）
```

### 4.3 详细端口清单

#### 4.3.1 监控服务器需要开放的入站端口

| 端口 | 协议 | 来源 | 目标服务 | 用途 | 是否必须 |
|------|------|------|----------|------|----------|
| **1157** | TCP | 运维终端/浏览器 | HertzBeat | 监控平台控制台 | ✅ 必须 |
| **3000** | TCP | 运维终端/浏览器 | Grafana | 日志可视化面板 | ✅ 必须 |
| 3100 | TCP | 应用服务器Promtail | Loki | 接收远程日志推送 | ✅ 必须（多服务器场景） |
| 3306 | TCP | 仅本机 | MySQL | 数据库连接 | ❌ 仅本机访问 |

#### 4.3.2 监控服务器需要开放的出站端口

| 端口 | 协议 | 目标 | 用途 | 是否必须 |
|------|------|------|------|----------|
| **应用端口（如8080/9090/8081）** | TCP | 各应用服务器 | SpringBoot Actuator / HTTP探针监控 | ✅ 必须 |
| **目标服务端口** | TCP | 各应用服务器 | TCP端口存活检测 | ✅ 必须 |
| **22** | TCP | 各应用服务器 | SSH进程/OS监控 | 可选 |
| **JMX端口（如9999）** | TCP | 各应用服务器 | JMX JVM深度监控 | 可选 |
| **443** | HTTPS | 企微/钉钉/飞书API | 告警Webhook通知 | 可选 |
| **465/25** | TCP | SMTP邮件服务器 | 邮件告警通知 | 可选 |

#### 4.3.3 应用服务器需要开放的入站端口

| 端口 | 协议 | 来源 | 目标服务 | 用途 | 是否必须 |
|------|------|------|----------|------|----------|
| **应用服务端口（如8080）** | TCP | 监控服务器HertzBeat | Spring Boot Actuator | 指标采集 | ✅ 必须 |
| **22** | TCP | 监控服务器HertzBeat | SSH | 进程/OS监控 | 可选 |
| **JMX端口（如9999）** | TCP | 监控服务器HertzBeat | JMX | JVM深度监控 | 可选 |

#### 4.3.4 应用服务器需要开放的出站端口

| 端口 | 协议 | 目标 | 用途 | 是否必须 |
|------|------|------|------|----------|
| **3100** | TCP | 监控服务器Loki | Promtail推送日志 / LokiLogbackAppender直发日志 | ✅ 必须（多服务器场景） |

### 4.7 多服务器 Promtail 部署方案

在每台应用服务器上部署独立的 Promtail 实例，采集本机日志后远程推送到监控服务器的 Loki。

#### 4.7.1 应用服务器 Promtail Docker 部署

在每台应用服务器上创建 `/opt/promtail/docker-compose.yml`：

```yaml
version: "3.8"

services:
  promtail:
    image: grafana/promtail:latest
    container_name: promtail
    volumes:
      - ./config.yml:/etc/promtail/config.yml
      - /var/log/app:/var/log/app:ro
    command: -config.file=/etc/promtail/config.yml
    restart: unless-stopped
```

创建 `/opt/promtail/config.yml`：

```yaml
server:
  http_listen_port: 9080
  grpc_listen_port: 0

positions:
  filename: /tmp/positions.yaml

clients:
  - url: http://10.x.1.10:3100/loki/api/v1/push

scrape_configs:
  - job_name: project-a-app-logs
    static_configs:
      - targets:
          - localhost
        labels:
          job: app-logs
          project: project-a
          server: 10.x.1.20
          environment: prod
          __path__: /var/log/app/*.log
    pipeline_stages:
      - regex:
          expression: '^\[(?P<level>DEBUG|INFO|WARN|ERROR)\]\s+(?P<timestamp>\S+)\s+(?P<logger>\S+)\s+-\s+(?P<message>.*)$'
      - labels:
          level:
      - timestamp:
          source: timestamp
          format: RFC3339
```

> **注意**：`clients.url` 必须填写监控服务器的实际 IP 地址，不能使用 `localhost`。

启动：

```bash
cd /opt/promtail
docker compose up -d
docker compose logs -f
```

#### 4.7.2 应用服务器 Promtail 二进制部署（无 Docker 环境）

```bash
# 下载 Promtail
cd /opt
wget https://github.com/grafana/loki/releases/latest/download/promtail-linux-amd64.zip
unzip promtail-linux-amd64.zip
chmod +x promtail-linux-amd64

# 创建配置文件（同上 4.7.1 的 config.yml）
mkdir -p /opt/promtail
vi /opt/promtail/config.yml

# 创建 systemd 服务
sudo tee /etc/systemd/system/promtail.service <<-'EOF'
[Unit]
Description=Promtail Log Shipper
After=network.target

[Service]
Type=simple
ExecStart=/opt/promtail-linux-amd64 -config.file=/opt/promtail/config.yml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 启动
sudo systemctl daemon-reload
sudo systemctl start promtail
sudo systemctl enable promtail
sudo systemctl status promtail
```

| 序号 | 建议 | 说明 |
|------|------|------|
| 1 | 限制源IP | 防火墙规则仅允许指定IP/网段访问，避免 0.0.0.0/0 |
| 2 | MySQL仅本机 | MySQL 3306 端口绑定 127.0.0.1，禁止远程访问 |
| 3 | Loki限制源IP | Loki 3100 端口仅允许应用服务器IP段访问 |
| 4 | SSH专用账号 | 创建只读监控账号，禁止root直接SSH |
| 5 | Actuator安全 | Spring Boot Actuator 端点配置访问认证或限制IP |
| 6 | HTTPS加密 | 生产环境 HertzBeat 和 Grafana 配置 HTTPS |
| 7 | 定期审计 | 定期检查防火墙规则，清理过期策略 |

## 五、被监控应用接入配置

### 5.1 Spring Boot 应用接入监控

#### 4.1.1 添加 Actuator 依赖

在被监控的 Spring Boot 应用的 `pom.xml` 中添加：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
```

#### 4.1.2 配置 Actuator 端点

在应用的 `application.yml` 中添加：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  endpoint:
    health:
      show-details: always
  metrics:
    tags:
      application: ${spring.application.name}
```

#### 4.1.3 在 HertzBeat 中添加监控

1. 登录 HertzBeat 控制台
2. 进入 **监控 → 新增监控**
3. 监控类型选择 **SpringBoot2** 或 **SpringBoot3**
4. 填写目标应用信息：
   - 名称：`项目A-用户服务`
   - 目标Host：`10.x.x.x`
   - 端口：`8080`
   - Actuator路径：`/actuator`
5. 选择所属项目（多项目管理功能）
6. 点击 **检测** 验证连通性
7. 点击 **确定** 添加监控

### 5.2 应用日志接入 Loki

#### 4.2.1 方式一：Logback Loki Appender（日志直发）

在应用的 `pom.xml` 中添加 HertzBeat 日志模块依赖：

```xml
<dependency>
    <groupId>org.apache.hertzbeat</groupId>
    <artifactId>hertzbeat-log</artifactId>
    <version>${hertzbeat.version}</version>
</dependency>
```

在应用的 `logback-spring.xml` 中添加 Loki Appender：

```xml
<configuration>
    <appender name="LOKI" class="org.apache.hertzbeat.log.loki.LokiLogbackAppender">
        <lokiUrl>http://<Loki服务器IP>:3100</lokiUrl>
        <projectName>项目A</projectName>
        <serviceName>用户服务</serviceName>
        <environment>prod</environment>
        <batchSize>100</batchSize>
        <flushIntervalMs>5000</flushIntervalMs>
    </appender>

    <root level="INFO">
        <appender-ref ref="LOKI"/>
        <!-- 保留原有的控制台或文件输出 -->
        <appender-ref ref="CONSOLE"/>
    </root>
</configuration>
```

#### 4.2.2 方式二：Promtail 文件采集（无需改代码）

1. 在应用服务器上部署 Promtail
2. 应用日志输出到文件，如 `/var/log/app/project-a-user-service.log`
3. Promtail 监控该目录，自动采集并发送到 Loki

Promtail 配置示例：

```yaml
server:
  http_listen_port: 9080
  grpc_listen_port: 0

positions:
  filename: /tmp/positions.yaml

clients:
  - url: http://<Loki服务器IP>:3100/loki/api/v1/push

scrape_configs:
  - job_name: project-a
    static_configs:
      - targets:
          - localhost
        labels:
          job: app-logs
          project: project-a
          environment: prod
          __path__: /var/log/app/project-a-*.log
    pipeline_stages:
      - match:
          selector: '{job="app-logs"}'
          stages:
            - regex:
                expression: '^\[(?P<level>DEBUG|INFO|WARN|ERROR)\]'
            - labels:
                level:
```

### 5.3 HTTP 接口监控接入

1. 进入 **监控 → 新增监控**
2. 监控类型选择 **Website**
3. 填写目标URL：`http://10.x.x.x:8080/api/health`
4. 配置检测间隔（建议 60 秒）
5. 配置告警规则：响应时间 > 3s 或 状态码 != 200 时告警

### 5.4 TCP 端口监控接入

1. 进入 **监控 → 新增监控**
2. 监控类型选择 **Port Telnet**
3. 填写目标Host和端口：`10.x.x.x:8080`
4. 配置检测间隔（建议 60 秒）
5. 配置告警规则：端口不可达时告警

## 七、常见问题排查

### 7.1 HertzBeat 启动失败

**现象**：HertzBeat 进程启动后立即退出

**排查步骤**：

```bash
# 查看启动日志
sudo journalctl -u hertzbeat --no-pager -n 100

# 常见原因：
# 1. MySQL 未启动或连接失败
docker compose ps mysql
docker exec mysql mysql -uhertzbeat -p<密码> -e "SELECT 1"

# 2. 端口被占用
netstat -tlnp | grep 1157

# 3. JDK 版本不匹配
java -version  # 需要 JDK 17+
```

### 7.2 监控目标连接失败

**现象**：添加监控后状态显示 DOWN

**排查步骤**：

```bash
# 1. 检查网络连通性
ping <目标IP>
telnet <目标IP> <端口>

# 2. 检查 Actuator 端点是否可访问
curl http://<目标IP>:<端口>/actuator/health

# 3. 检查防火墙
sudo firewall-cmd --list-ports

# 4. 检查目标应用 Actuator 配置
# 确认 management.endpoints.web.exposure.include 包含 health,metrics
```

### 7.3 告警通知未发送

**现象**：告警触发但未收到通知

**排查步骤**：

```bash
# 1. 检查通知接收人配置
# 在 HertzBeat 控制台 → 告警通知 → 通知接收人 中确认配置

# 2. 检查通知策略
# 确认策略已启用，且标签匹配告警标签

# 3. 检查 Webhook URL 是否可达
curl -X POST "<企业微信WebhookURL>" \
  -H "Content-Type: application/json" \
  -d '{"msgtype":"text","text":{"content":"测试消息"}}'

# 4. 检查 HertzBeat 日志
sudo journalctl -u hertzbeat | grep -i "notify\|alert\|notice"
```

### 7.4 Loki 日志查询为空

**现象**：Loki 查询无结果

**排查步骤**：

```bash
# 1. 检查 Promtail 是否运行
docker compose ps promtail
docker compose logs promtail

# 2. 检查日志文件是否存在
ls -la /opt/hertzbeat/data/logs/

# 3. 检查 Promtail positions 文件
docker exec promtail cat /tmp/positions.yaml

# 4. 检查 Loki 是否接收到数据
curl "http://localhost:3100/loki/api/v1/labels"
curl "http://localhost:3100/loki/api/v1/label/job/values"

# 5. 检查 Promtail 到 Loki 的网络连通性
docker exec promtail wget -qO- http://loki:3100/ready
```

### 7.5 MySQL 连接数不足

**现象**：日志中出现 "Too many connections"

**解决方法**：

```bash
# 增加 MySQL 最大连接数
docker exec mysql mysql -uroot -p<Root密码> -e "SET GLOBAL max_connections = 500"

# 永久生效：在 docker-compose.yml 的 mysql command 中添加
command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci --max-connections=500
```

### 7.6 磁盘空间不足

**现象**：服务异常，日志报磁盘空间不足

**解决方法**：

```bash
# 检查磁盘使用
df -h

# 清理 Docker 无用资源
docker system prune -a

# 清理旧日志
find /opt/hertzbeat/data/logs -name "*.log" -mtime +30 -delete

# 配置 Loki 日志保留期（在 Loki 配置中添加）
# table_manager:
#   retention_deletes_enabled: true
#   retention_period: 720h  # 30天
```

## 附录 A：一键部署脚本

将以下脚本保存为 `/opt/hertzbeat/deploy.sh`，赋予执行权限后运行：

```bash
#!/bin/bash
set -e

DEPLOY_DIR="/opt/hertzbeat"

echo "===== 1. 启动基础设施 ====="
cd $DEPLOY_DIR
docker compose up -d

echo "===== 2. 等待 MySQL 就绪 ====="
for i in {1..30}; do
  if docker exec mysql mysql -uhertzbeat -phertzbeat123 -e "SELECT 1" &>/dev/null; then
    echo "MySQL 已就绪"
    break
  fi
  echo "等待 MySQL 启动... ($i/30)"
  sleep 2
done

echo "===== 3. 启动 HertzBeat ====="
sudo systemctl restart hertzbeat

echo "===== 4. 等待 HertzBeat 就绪 ====="
for i in {1..60}; do
  if curl -s http://localhost:1157/actuator/health | grep -q "UP"; then
    echo "HertzBeat 已就绪"
    break
  fi
  echo "等待 HertzBeat 启动... ($i/60)"
  sleep 3
done

echo "===== 5. 服务状态 ====="
docker compose ps
sudo systemctl status hertzbeat --no-pager

echo "===== 部署完成 ====="
echo "HertzBeat 控制台：http://$(hostname -I | awk '{print $1}'):1157"
echo "Grafana 面板：http://$(hostname -I | awk '{print $1}'):3000"
echo "默认账号：admin / hertzbeat"
```
