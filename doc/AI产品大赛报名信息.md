# AI产品大赛报名 - 作品信息

## 作品名称

**Java微服务智能运维监控中台**

> 备选名称：
> - 基于HertzBeat的微服务智能监控告警平台
> - 政务外网多项目统一运维监控平台
> - 智能化微服务可用性监测与日志分析中台

---

## 应用场景

面向**政务外网及企业内网**的Java微服务集群运维场景，解决多项目多服务的**统一监控、异常发现、告警通知、日志排查**四大核心痛点：

1. **多项目统一运维**：多个政务/业务系统分散在不同服务器，缺乏统一的监控视角和项目管理能力，运维人员需要在多个系统间切换排查问题。
2. **服务异常感知滞后**：Spring Cloud/Spring Boot 微服务出现假死、宕机、启动失败等故障时，无法第一时间发现，依赖用户报障后被动响应，影响业务连续性。
3. **告警风暴与遗漏**：传统监控告警规则单一，缺少多级告警、告警升级、告警抑制机制，导致重要告警被淹没或无人处理。
4. **日志排查效率低下**：各应用日志分散在各自服务器上，缺乏统一的采集、存储、检索能力，故障排查时需逐台登录服务器查看日志，耗时耗力。

---

## 作品简介

基于**Apache HertzBeat**开源监控框架进行二次开发，融合**Grafana Loki**日志聚合技术，构建了一套面向Java微服务集群的**智能化统一运维监控中台**。平台通过**无Agent主动采集**方式，对Spring Boot/Spring Cloud服务的Actuator端点、HTTP接口、TCP端口、SSH进程等多维度指标进行实时监测；内置**多级告警引擎**，支持告警自动升级、抑制去重、静默维护等智能策略；集成**统一日志管理**体系，通过LokiLogbackAppender实现应用日志直发集中存储，支持按项目、服务、级别等多维度检索与告警关联跳转；提供**多项目维度**的组织管理能力和**运维总览Dashboard**，实现从单服务健康检测到全局SLA可用性分析的全方位运维可视化。平台采用Docker Compose一键部署，适配政务外网离线环境，并通过精细化的网络端口规划支持跨服务器分布式监控与日志采集，显著提升运维团队的问题发现速度和故障排查效率。

## 核心亮点（补充说明，如需要）

| 亮点 | 说明 |
|------|------|
| **零侵入采集** | 无需在被监控服务安装Agent，通过HTTP/JMX/SSH协议主动采集，对业务零侵入 |
| **智能告警链路** | 多级告警（紧急/严重/警告）+ 自动升级 + 抑制去重 + 维护静默，避免告警风暴 |
| **SLA可用性量化** | 自动计算每个服务的月度SLA可用率，连续失败阈值判定服务降级/不可用状态 |
| **日志-告警联动** | 告警详情页一键跳转到对应时间段的日志检索页面，缩短MTTR（平均恢复时间） |
| **多项目隔离** | 按项目维度组织监控目标、告警规则和通知策略，适合多团队多项目并行运维 |
| **政务外网适配** | 支持离线镜像部署、精细化防火墙策略、Systemd服务托管，满足政务外网安全要求 |
| **开源二次开发** | 基于Apache 2.0协议开源产品二次开发，技术栈（Java/SpringBoot）与现有项目完全一致 |
